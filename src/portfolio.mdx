## 조회 작업의 개선과 성능 튜닝

E2E 테스트 중 원서 검색 API가 체감될 정도로 느리다는 피드백을 받았습니다.

불필요한 20여 개 row가 조회되던 쿼리는 재사용 가능성도 낮아, 프로젝션 기반 전용 쿼리로 개선하였습니다.  
추가로 세컨더리 인덱스, Redis 캐시를 도입해 <span class = "bold">550ms → 30ms, 약 95% 성능 향상을 이끌었습니다.</span>

## 원서 수정 기능 트러블 슈팅 및 데이터 모델링 개선

- EDA 아키텍처로 도메인 간 결합도를 낮추고, 이벤트 기반 트랜잭션과 보상 로직으로 결과적 일관성을 유지했습니다.

## Spring Batch 퍼모먼트 향상

베팅 정산 처리 로직은 Spring Batch를 사용해 개발되어 있었습니다.  
하지만 기존 구조에서 대량의 데이터를 처리하는 상황에 큰 성능 저하가 있는 것을 확인했습니다.  
다양한 테스트 시나리오를 통해 병목 지점을 확인하였고, 이를 기반으로 다음과 같은 성능 개선 작업을 하였습니다.

- ItemProcessor를 제거하여, 청크 단위 일괄 처리 구조로 개선
- Paging 처리의 성능 저하 문제를 해결하기 위해, Cursor 기반 Reader 적용
- 배치 로직에서 JPA를 걷어내고, Exposed DSL을 활용해 Batch Insert 적용

그 결과, 100만 건 처리 기준으로 <span class = "bold">30분 -> 1분 내외로 단축</span>, <span class = "bold">약 30배 성능을 향상</span>시켰습니다.

# Skills

- 백엔드 엔지니어로서, 해결해야 하는 문제를 기술적으로 풀어낼 수 있도록 탐구합니다.
- 문제를 해결하기 위해 새로운 기술을 빠르게 습득하고 활용할 수 있습니다.
- 심도 깊은 코드리뷰와 피드백을 주고받으며 코드의 품질을 높이는 문화를 좋아합니다.
- 직위와 포지션에 상관없이 자유롭게 의견을 나눌 수 있는 분위기를 선호합니다.

## Java & Kotlin

- jvm에 대한 기본적인 이해가 있습니다.
  - 클래스로더, JIT 등의 동작 과정을 이해하며 성능 개선을 위해 low-level 관점에서도 접근할 수 있습니다.
- java 8 이후에 도입된 기능들을 적극적으로 활용할 수 있습니다.
- java reflection을 이해하고 spring frameworks를 사용합니다.
  - DI, Proxy, 어노테이션 프로세싱의 내부 동작을 이해하고 이를 직접 구현하거나 활용한
    경험이 있습니다.
- java 프로젝트를 kotlin 100%로 마이그레이션한 경험이 있으며, null-safety 등의 기능을 활용할 수 있습니다.

## springboot

- DI, IoC, Bean 등의 스프링 기본 원리에 대해 이해하고 있습니다.
- 스프링 트랜잭션에 대해 이해하고 있으며, 지원하는 기능을 활용할 수 있습니다.
- 도메인 요구사항을 토대로 Restful API를 개발할 수 있습니다.

## JPA

- 엔티티와 영속성 컨텍스트에 대해 이해하고 있습니다.
- JPA의 내부 동작 과정과 JDBC, 커넥션 풀 등 데이터베이스 접근 기술에 대해 이해하고 있습니다.
- N+1 문제, OSIV, 2차 캐시, JPQL 등 JPA에서 발생할 수 있는 이슈와 최적화 방법을 숙지하고 있습니다.
- QueryDSL로 type-safety한 쿼리를 작성하며, 페이징, Batch Size 조정 등으로 성능 개선 경험이 있습니다.

## DB

- RDMS 개념을 이해하며 주로 MySQL을 사용합니다.
- 비즈니스 요구사항을 바탕으로 ERD 설계, 정규화 등으로 데이터 모델링을 할 수 있습니다.
- MySQL의 인덱스 자료구조 개념, 트랜잭션 동작 과정의 이해가 있으며 성능 개선에 활용할 수 있습니다.

## Redis

- Redis의 기본 개념의 이해가 있습니다.
- 다양한 자료구조를 상황에 맞게 사용할 수 있습니다.
- LFU, LRU 메모리 관리 정책과, maxmemory 설정 등으로 최적화 경험이 있습니다.
- 고가용성 구조를 위한 복제, 센티널, 클러스터 구조에 대한 기본적인 이해가 있습니다.

## DevOps

- 시스템 자동화에 관심이 많으며, 배포 파이프라인에 대한 기본적인 이해가 있습니다.
- Docker 컨테이너에 대해 이해하고 활용할 수 있습니다.
- github actions 기반 CI/CD를 구현할 수 있습니다.
- AWS의 기능을 이해하고 활용할 수 있으며, Cloud Watch로 application 모니터링을 구축한 경험이 있습니다.
